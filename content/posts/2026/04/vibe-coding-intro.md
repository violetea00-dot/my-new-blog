## 바이브 코딩이란?

최근 개발자 커뮤니티에서 **바이브 코딩(Vibe Coding)** 이라는 표현이 자주 등장하고 있습니다. 이는 AI 어시스턴트와 자연스러운 대화를 통해 코드를 작성해 나가는 개발 방식을 말합니다. 직접 모든 코드를 타이핑하는 것이 아니라, 원하는 것을 말하고 AI가 제안한 코드를 검토하면서 방향을 잡아가는 방식입니다.

## 기존 개발 방식과의 차이

| 기존 방식 | 바이브 코딩 |
|-----------|-------------|
| 직접 코드 작성 | AI 제안 → 검토 → 수정 |
| 문서 검색 후 구현 | 자연어로 요청 |
| 에러 → 구글링 | 에러 → AI에게 설명 요청 |
| 혼자 고민 | AI와 페어 프로그래밍 |

## 실제로 써보니

GitHub Copilot을 활용해 이 블로그를 만들면서 느낀 점을 솔직하게 공유합니다.

### 좋았던 점

- **반복 작업 감소**: 컴포넌트 구조, 타입 정의, 보일러플레이트 코드 등 반복적인 부분을 빠르게 처리할 수 있었습니다.
- **새로운 라이브러리 학습**: `rehype-pretty-code`, `gray-matter` 같이 처음 써보는 라이브러리도 AI 설명을 들으며 빠르게 적용할 수 있었습니다.
- **에러 해결**: 빌드 오류가 발생했을 때 에러 메시지를 그대로 붙여넣으면 원인과 해결 방법을 바로 알 수 있었습니다.

### 주의할 점

- AI가 제안하는 코드를 **무조건 믿어서는 안 됩니다**. 항상 무엇을 하는 코드인지 이해하고 적용해야 합니다.
- 간혹 라이브러리 버전이 맞지 않거나 deprecated된 방식을 제안하는 경우가 있습니다.
- 결국 최종 판단은 **개발자 본인**이 해야 합니다.

## 바이브 코딩을 잘 활용하는 법

1. **목표를 명확히 설명하기** — "버튼 만들어줘" 보다 "다크모드를 토글하는 sun/moon 아이콘 버튼 만들어줘"처럼 구체적으로 요청할수록 좋은 결과가 나옵니다.
2. **작은 단위로 나눠 요청하기** — 한 번에 너무 많은 것을 요청하면 결과물이 복잡해집니다.
3. **이해하면서 진행하기** — 모르는 코드가 나오면 바로 설명을 요청하세요.
4. **직접 수정도 두려워하지 않기** — AI 제안이 완벽하지 않을 때는 직접 고치는 것이 빠릅니다.

## 마치며

바이브 코딩은 개발자를 대체하는 것이 아니라, **개발자의 생산성을 높여주는 도구**입니다. AI와 함께 작업하면서 오히려 더 많은 것을 시도해볼 수 있게 되었고, 이 블로그도 그 결과 중 하나입니다.

앞으로도 AI 도구를 활용한 개발 경험을 꾸준히 기록해 나갈 예정입니다.
